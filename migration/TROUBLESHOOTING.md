# 문제 해결 가이드

변환 도구 사용 중 발생할 수 있는 문제와 해결 방법입니다.

## 자주 발생하는 문제

### 1. "'입력란' 시트를 찾을 수 없습니다" 에러
- **원인**: Excel 파일의 시트 이름이 '입력란'이 아니거나 시트 자체가 없는 경우입니다.
- **해결 방법**: 
    - 해당 엑셀 파일을 열어 시트 이름을 확인하세요. 
    - 표준 양식을 따르지 않는 오래된 파일일 수 있습니다. 수동으로 시트 이름을 '입력란'으로 수정하거나 해당 파일은 제외하고 진행하세요.

### 2. "No sheet named '제품표준서'" 경고
- **원인**: 저장방법, 유통기한 정보가 들어있는 '제품표준서' 시트를 찾지 못했습니다.
- **해결 방법**: 
    - 프로그램이 '제품표준서' 외에도 공백이 포함된 이름을 자동으로 찾아보지만, 이름이 완전히 다른 경우에는 데이터를 가져올 수 없습니다. 
    - 이 경우 기본 정보는 정상 추출되나, 저장방법과 유통기한 항목이 비어 있게 됩니다.

### 3. 결과 CSV 파일에서 한글이 깨짐
- **원인**: 엑셀로 CSV 파일을 열었을 때 인코딩 인식 문제로 깨져 보일 수 있습니다.
- **해결 방법**: 
    - 이 도구는 자동으로 'UTF-8 with BOM (UTF-8-SIG)' 인코딩으로 저장하므로 엑셀에서 바로 열어도 깨지지 않아야 합니다.
    - 만약 깨진다면, 엑셀의 [데이터] -> [텍스트/CSV에서] 메뉴를 통해 인코딩을 '65001: 유니코드(UTF-8)'로 설정하여 불러오세요.

### 4. 중복된 제품 코드 (`duplicates_` 파일 생성)
- **원인**: 여러 개의 엑셀 파일이 동일한 '제품코드'를 가지고 있습니다.
- **해결 방법**: 
    - `duplicates_products.csv` 파일을 열어 '원본파일' 컬럼을 확인하세요. 
    - 어떤 파일이 더 최신 데이터인지 판단한 후, 불필요한 원본 엑셀 파일을 삭제하거나 다른 폴더로 옮기고 다시 실행하세요.

### 5. "Python이 설치되어 있지 않습니다" 에러
- **원인**: 시스템에 Python이 설치되어 있지 않거나 환경 변수(PATH)에 등록되지 않았습니다.
- **해결 방법**: 
    - Python을 다시 설치하고 설치 옵션 중 "Add Python to PATH"를 반드시 체크하세요.
    - 터미널(cmd)에서 `python --version`을 입력했을 때 버전 정보가 나오는지 확인하세요.

### 6. "ModuleNotFoundError: No module named 'xlrd'"
- **원인**: 필요한 라이브러리가 설치되지 않았습니다.
- **해결 방법**: 
    - `run.bat`를 실행하면 자동으로 설치를 시도합니다. 
    - 인터넷 연결을 확인하고 다시 실행하거나, 터미널에서 `pip install -r requirements.txt`를 직접 입력하세요.
